AM VM Docker Remote API 설정 검증 수정 요약

작성일: 2026-07-07 · 대상: DockerSetupService · 다음 릴리즈 배포 후 Azure VM 검증 필요
문제의 본질
Remote API 설정 실패를 기존 dockerd 상태로 성공 오판 가능
수정 방향
Docker 설치 여부와 Remote API 설정 여부를 분리
지원 범위
systemd docker.service 기반 Docker만 자동 설정

한 줄 결론

Docker가 이미 있으면 재설치하지 않고, Remote API만 systemd 방식으로 설정한다. 이때 docker.service가 없거나 systemctl restart docker가 실패하면 성공으로 넘기지 않고 실패 처리한다.

배포 경로

준비 단계

1
AM이 Tumblebug cmd/infra API로 VM 명령 실행
2
VM에서 docker --version과 dockerd -H tcp://0.0.0.0:2375 확인
3
Remote API가 없으면 systemd docker.service 방식으로 설정 시도

실제 컨테이너 배포 단계

1
AM이 VM Public IP 조회
2
Docker client가 tcp://{VM_PUBLIC_IP}:2375로 접속
3
Docker Remote API로 이미지 pull, 컨테이너 생성, 컨테이너 시작 수행

상태별 처리 결과

VM 상태 AM 동작 판정
Docker 없음 get-docker.sh 설치 후 systemd 기반 Remote API 설정 기존 설치 흐름 유지
Docker 있음 + Remote API 있음 설치, 설정, restart 없이 skip 배포 가능 상태로 인정
Docker 있음 + Remote API 없음 Docker 재설치 없이 Remote API 설정만 시도 이번 수정의 핵심 분기
Remote API 설정 중 docker.service 없음 override.conf 작성 전에 실패 처리 지원 불가 VM 상태
systemctl restart docker 실패 최종 dockerd 검증으로 넘어가지 않고 실패 처리 설정 적용 실패

코드 변경 포인트

1. 설치 여부와 Remote API 여부 분리

Docker version과 Remote API enabled를 별도 boolean으로 판단한다.

dockerInstalled remoteApiEnabled

2. 기존 Docker 재설치 방지

Docker가 이미 있으면 installDocker()를 다시 호출하지 않고 configureInstalledDockerOnVM()으로 이동한다.

3. systemd 지원 범위 명확화

Remote API 설정 전 systemctl cat docker.service로 systemd 관리 대상인지 확인한다.

4. 실패 마커 검사

override 작성, daemon reload, Docker restart 결과를 마커로 검사해 실패를 숨기지 않는다.

성공/실패 마커

단계 성공 마커 실패 마커
systemd docker.service 확인 DOCKER_SERVICE_FOUND DOCKER_SERVICE_NOT_FOUND
Remote API override 작성 DOCKER_REMOTE_CONFIGURED DOCKER_REMOTE_CONFIG_FAILED
Docker restart DOCKER_RESTARTED DOCKER_RESTART_FAILED

릴리즈 후 테스트 체크리스트

정상 Azure VM 신규 배포

  • VM spec: Standard_D2als_v6
  • 테스트 앱: nginx 등 단순 컨테이너
  • 로그: DOCKER_SERVICE_FOUND, DOCKER_REMOTE_CONFIGURED, DOCKER_RESTARTED
  • 결과: AM 배포 상태 SUCCESS 또는 Running 계열

이미 Remote API가 켜진 VM

  • 초기 체크에 Docker version과 Remote API enabled 포함
  • 설치/restart 없이 skip
  • 기존 배포 성공 동작 유지

docker.service 없는 VM

  • Docker systemd service is not available 메시지 확인
  • override.conf 작성 단계로 진행하지 않음
  • 성공으로 오판하지 않음

restart 실패 VM

  • Failed to restart Docker service after remote API configuration 메시지 확인
  • 기존 dockerd 상태만 보고 성공 처리하지 않음
  • 실패 원인을 AM 로그에서 식별 가능

주의 사항

Docker Remote API 2375는 TLS 없는 평문 API다. 운영 환경에서는 AM 서버에서만 접근 가능하도록 CSP 보안그룹 또는 방화벽으로 제한해야 한다.

이번 수정은 docker.service가 없는 모든 Docker 실행 방식을 지원하는 변경이 아니다. AM이 지원하는 systemd Docker 전제를 명확히 하고, 그 전제 밖이면 성공으로 오판하지 않도록 하는 변경이다.
